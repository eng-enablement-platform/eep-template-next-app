Most templates stop at "deploy to Vercel". This guide takes you the rest of the
way — a real domain, proper DNS, and a production Clerk instance that works with it.
The full stack is: **Cloudflare** (registrar + DNS) → **Vercel** (hosting) → **Clerk**
(auth). All three have free tiers.

---

## Step 1 — Buy a domain on Cloudflare

[Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) sells domains at
cost (no markup) and includes free DNS management, DDoS protection, and SSL — making it
the best value registrar for this stack.

1. Go to [dash.cloudflare.com](https://dash.cloudflare.com) and create an account
2. Navigate to **Domain Registration** → **Register Domains**

![Cloudflare domain search results](/cloudflare/cloudflare-buy-domain.png)

3. Search for your domain — you'll see availability and pricing

![Cloudflare domain purchase form](/cloudflare/cloudflare-purchase-domain.png)

4. Complete the purchase — Cloudflare will confirm and activate the domain

![Cloudflare domain confirmation](/cloudflare/cloudflare-domain-confirmation.png)

<Callout type='idea'>
  `.dev` domains are a natural fit for developer tools — they're Google-owned,
  require HTTPS by default (browsers won't load them without SSL), and signal
  the purpose of the site immediately. `yourproject.dev` is a good pattern.
</Callout>

<Callout type='info'>
  Once purchased, Cloudflare automatically becomes your DNS provider. You'll
  manage all DNS records from the Cloudflare dashboard.
</Callout>

## Step 2 — Connect your domain to Vercel

Vercel needs to know about your domain before it can serve traffic on it.

<CollapsibleSteps>

<CollapsibleStep title="Add the domain to your Vercel project" defaultOpen={false}>

1. Open your project in the [Vercel Dashboard](https://vercel.com/dashboard)
2. Go to **Settings** → **Domains**
3. Enter your domain (e.g. `yourproject.dev`) and click **Add**
4. Vercel will show you the DNS records to add — copy them, you'll need them in
   the next step

For a typical setup you'll get:

| Record type | Name       | Value                  |
| ----------- | ---------- | ---------------------- |
| `A`         | `@` (apex) | Vercel's IP address    |
| `CNAME`     | `www`      | `cname.vercel-dns.com` |

</CollapsibleStep>

<CollapsibleStep title="Add the DNS records in Cloudflare" defaultOpen={false}>

1. In the Cloudflare dashboard, select your domain
2. Go to **DNS** → **Records** → **Add record**
3. Add the `A` record for the apex domain (`@`)
4. Add the `CNAME` record for `www`

<Callout type='warn'>
  Cloudflare's proxy (the orange cloud icon) intercepts traffic before it
  reaches Vercel. For Vercel deployments, set the proxy status to **DNS only**
  (grey cloud) on the records you add — otherwise Vercel's SSL certificate
  provisioning will fail. You can re-enable Cloudflare's proxy later once
  everything is working.
</Callout>

DNS propagation typically takes a few minutes with Cloudflare, occasionally up to
an hour. Vercel will show a green checkmark in the Domains settings once it detects
the records and provisions an SSL certificate.

</CollapsibleStep>

<CollapsibleStep title="Add the docs subdomain (optional)" defaultOpen={false}>

If you're deploying the docs site as a separate Vercel project (recommended),
add a `docs` subdomain:

1. In your **docs** Vercel project → **Settings** → **Domains** → add
   `docs.yourproject.dev`
2. Vercel gives you a `CNAME` record pointing to `cname.vercel-dns.com`
3. Add that `CNAME` in Cloudflare DNS with the `docs` name, proxy status **DNS only**

</CollapsibleStep>

</CollapsibleSteps>

## Step 3 — Update Clerk for your production domain

Clerk needs to know your production domain to validate auth requests and redirect
correctly after sign-in.

<CollapsibleSteps>

<CollapsibleStep title="Add your domain to the Clerk production instance" defaultOpen={false}>

Clerk Dashboard → **Configure** → **Domains** → **Add domain** → enter
`yourproject.dev`.

Clerk will verify ownership by asking you to add a DNS record. Add the record in
Cloudflare, wait for Clerk to verify it, then set it as the primary domain.

</CollapsibleStep>

<CollapsibleStep title="Update the Clerk redirect URLs in Vercel" defaultOpen={false}>

Now that your domain is live, update the Clerk redirect env vars in your Vercel
project to use your real domain rather than the `.vercel.app` URL. In Vercel →
**Settings** → **Environment Variables** (Production environment):

| Variable                              | Value      |
| ------------------------------------- | ---------- |
| `NEXT_PUBLIC_CLERK_SIGN_IN_URL`       | `/sign-in` |
| `NEXT_PUBLIC_CLERK_SIGN_UP_URL`       | `/sign-up` |
| `NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL` | `/`        |
| `NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL` | `/`        |

These are path-relative and don't need the domain — Clerk resolves them against
the current origin automatically.

</CollapsibleStep>

<CollapsibleStep title="Update the GCP OAuth redirect URI (if using Google SSO)" defaultOpen={false}>

If you set up Google SSO via GCP, the OAuth client has an authorised redirect URI
pointing to the old Clerk domain. You need to add your new domain's Clerk callback:

1. Go to [console.cloud.google.com](https://console.cloud.google.com) → **APIs &
   Services** → **Credentials** → your OAuth client
2. Under **Authorised redirect URIs**, add the new Clerk callback URL for your
   production domain. Find the exact URL in: Clerk Dashboard → **Configure** →
   **SSO Connections** → **Google** → **Redirect URI**
3. Save the GCP credential

</CollapsibleStep>

</CollapsibleSteps>

## Step 4 — Redeploy and verify

Trigger a fresh deployment in Vercel (or push a commit) so the build picks up
any env var changes. Then verify:

- `https://yourproject.dev` loads the app
- `https://docs.yourproject.dev` loads the docs (if set up)
- Sign-in and sign-up flows complete without errors
- Google SSO redirects correctly (if enabled)

<Callout>
  If sign-in fails after switching domains, the most likely cause is a stale
  Clerk domain config or a GCP redirect URI mismatch. Check Clerk Dashboard →
  **Configure** → **Domains** first, then the GCP OAuth client.
</Callout>

## DNS records summary

A complete Cloudflare DNS setup for this stack:

| Type            | Name               | Value                  | Proxy    | Purpose                   |
| --------------- | ------------------ | ---------------------- | -------- | ------------------------- |
| `A`             | `@`                | Vercel IP              | DNS only | Main app apex domain      |
| `CNAME`         | `www`              | `cname.vercel-dns.com` | DNS only | www redirect              |
| `CNAME`         | `docs`             | `cname.vercel-dns.com` | DNS only | Docs site                 |
| `TXT` / `CNAME` | _(Clerk-provided)_ | _(Clerk-provided)_     | DNS only | Clerk domain verification |
