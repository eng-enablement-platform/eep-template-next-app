## Prerequisites

| Tool    | Version | Notes                                                                                                                                             |
| ------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Node.js | ≥ 24    | Install via [nvm](https://github.com/nvm-sh/nvm) or [Homebrew](https://brew.sh/)                                                                  |
| pnpm    | ≥ 10    | `npm install -g pnpm`                                                                                                                             |
| Docker  | any     | [Docker Desktop](https://www.docker.com/products/docker-desktop/) or [Rancher Desktop](https://rancherdesktop.io/) (free, works on Apple Silicon) |

---

The repo ships an `.nvmrc` pinning the required Node version. Pick whichever
version manager you prefer:

```bash
# nvm (most common)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
nvm install  # reads .nvmrc and installs the pinned version
nvm use      # switch to it in the current shell
```

```bash
# Homebrew (macOS, installs latest stable — check .nvmrc for required version)
brew install node
```

## Environment variables

Copy the template and fill in your secrets:

```bash
cp .env.local_template .env.local
```

The key variables to set before the app will run:

| Variable                            | What it is                 | Where to get it                                                                   |
| ----------------------------------- | -------------------------- | --------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk public key           | [Clerk dashboard](https://dashboard.clerk.com) → API Keys                         |
| `CLERK_SECRET_KEY`                  | Clerk secret key           | [Clerk dashboard](https://dashboard.clerk.com) → API Keys                         |
| `DATABASE_URL`                      | Postgres connection string | Pre-filled for local Docker (`postgresql://postgres:postgres@localhost:5432/app`) |

The remaining variables (`NEXT_PUBLIC_CLERK_SIGN_IN_URL` etc.) are pre-filled in the template and work as-is for local dev.

See [Authentication Setup](/docs/dev-environment/authentication) for Clerk-specific steps and [Database](/docs/dev-environment/database) for DB setup.

## Start the database

```bash
docker compose up -d
```

Then run migrations (safe to re-run, skips already-applied ones):

```bash
pnpm db:migrate
```

Optionally seed demo data:

```bash
pnpm db:seed   # resets example_items to 3 fixed rows
```

## Dev servers

The repo contains two apps: the **Next.js app** (`:3000`) and the **docs site** (`:3001`).

| Command         | What starts                            |
| --------------- | -------------------------------------- |
| `pnpm dev`      | App only — `http://localhost:3000`     |
| `pnpm dev:docs` | Docs only — `http://localhost:3001`    |
| `pnpm dev:all`  | Both concurrently, colour-coded output |

<Callout>For most work you just need `pnpm dev`.</Callout>

## Useful commands

```bash
pnpm test          # unit tests (Vitest, single pass)
pnpm test:watch    # watch mode
pnpm typecheck     # tsc --noEmit
pnpm lint          # ESLint
pnpm e2e           # Playwright end-to-end (needs pnpm dev running)
pnpm build         # production build
```

<Callout type='info'>
  Prefer the [Dev Container](/docs/dev-environment/dev-container) if you want a
  zero-setup environment — it pre-installs everything and wires the database
  automatically.
</Callout>
